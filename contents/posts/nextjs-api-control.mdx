---
title: Next.js route handler에서 API 관리하면 어떨까?
description: think
date: 2024-05-06
---

API 관련 파일이나 폴더를 따로 만들지 않고 Next.js에서 제공하는 route handler에서 백엔드 API를 모두 관리하면 어떨까?

Next.js 정적으로 실행하는 것이 아니라면 어차피 서버를 하나 생성하게 되는 것인데(설마 서버를 사용하지 않으면 생성하지 않는 건 아니겠지?)

API 모킹도 가능하고 API 상태에 따라 데이터 처리도 가능하니 좋지 않을까?

백엔드 서버에 직접 요청할 수 있는 것을 Next.js 서버를 거쳐가는 것이 좋은 방법인지는 모르겠다.

각자의 상황마다 다르겠지만, 내 생각은 데이터를 모킹하는 정도로 라이브러리를 설치하지 않아도 되고 API를 route handler에서 모두 관리할 수 있어 route handler를 적극 사용하는 것이 좋을 것 같다.
