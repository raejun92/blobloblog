## CSS 사용 유형

### 일반 CSS

- 기본적으로 따로 설정할 필요 없이 사용할 수 있다.

---

### CSS 전처리기

- CSS 전처리기로 CSS 확장 버전이다. 변수 중첩, mixin, nesting 등을 지원하여 코드의 재사용성과 가독성을 높일 수 있다.
- 작성한 코드를 빌드 프로세스나 컴파일러를 통해 일반적인 CSS 파일로 변환하고, 이후 브라우저에서 해당 CSS 파일을 다운로드하여 적용한다.
- Sass, LESS, SCSS 등

---

### CSS 프레임워크

- 미리 디자인된 구성요소와 스타일을 제공하여 개발자가 프로젝트를 보다 쉽게 개발할 수 있도록 한다.
- 미리 정의한 디자인을 빌드 시에 프로젝트에 번들링되고, 브라우저가 해당 번들된 CSS 파일을 다운로드하여 적용한다.
- Bootstrap, Tailwind CSS 등

---

### CSS-in-JS

- JavaScript를 사용하여 컴포넌트 수준에서 스타일을 작성하는 방식이다. 이는 스타일을 캡슐화하고, 동적으로 스타일을 조작하며 컴포넌트 단위의 스타일을 관리하는 데 유용하다.
- 런타임 시 CSS 생성
  - JavaScript를 사용하여 컴포넌트를 렌더링하는 동안에 CSS가 동적으로 생성되어 컴포넌트와 함께 렌더링된다. 이로 인해 해당 컴포넌트가 렌더링될 때 스타일이 동적으로 생성된다.
- 빌드 시 CSS 생성
  - JavaScript를 컴파일하여 일반적인 CSS로 변환한다. JavaScript안에 CSS를 작성하고 이를 CSS로 컴파일하여 파일을 생성한다. 이 파일은 빌드 후에 브라우저에서 다운로드하여 적용된다.
- Styled Components, Emotion, vanilla-extract 등
