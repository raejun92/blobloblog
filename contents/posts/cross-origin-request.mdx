### 단순 요청(Simple Request)

- GET, POST, HEAD
- 안전한 헤더
  - Accept
  - Accept-Language
  - Content-Language
  - Content-Type

### 사전 요청(Preflighted Request)

- PUT, UPDATE
- 매번 사전요청을 보내면 단순 요청에 비해 비용이 많이 든다.
  - server 승인에 따라 browser에서 사전 요청을 생략할 수 있다. -> server가 다른 출처에 일정 기간 사전 요청을 캐시해도 된다.
  - OPTIONS 메소드는 특정 리소스에 대한 서버 자원 메서드 목록을 요청하여 클아이언트에게 허용된 작업을 확인하는 메소드이다.

### 출처

- browser는 요청을 보낼 때는 Origin을 보낸다.
- server는 access-controll-allow-origin에 허용할 교차 출처를 적어 보낸다.

### 메소드

- browser는 access-control-request-methods에 사용할 목록을 적어 보낸다.
- server는 access-control-allow-methods에 허용할 목록을 적어 보낸다.

### 헤더

- broswer는 access-control-request-headers에 안전하지 않은 헤더 목록을 적어 보낸다.
- server는 access-control-allow-headers에 사용해도 되는 헤더 목록을 적어 보낸다.

### 캐시

- server는 access-control-max-age에 캐시 시간을 적어 보낸다.
